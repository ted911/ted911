# GitHub 프로필 README 설정 가이드

이 문서는 개발자의 전문성을 효과적으로 드러낼 수 있는 GitHub 프로필 README를 작성하는 방법을 안내합니다. 제공된 키트(profile-kit)를 활용하여 쉽고 빠르게 포트폴리오 수준의 프로필을 완성해 보시기 바랍니다.

## 개요

GitHub 프로필은 동료 개발자에게 가장 먼저 보여지는 공간입니다. 이 가이드를 통해 다음과 같은 요소들을 설정할 수 있습니다.

* **상단 배너 (Typing SVG)**: 주요 직무와 역할을 동적으로 보여줍니다.
* **기술 스택 그리드 (Bento Grid)**: 보유 기술을 깔끔한 그리드 형태로 시각화합니다.
* **상세 기여 내역**: AI를 활용하여 프로젝트 경험을 전문적인 문구로 정리합니다.
* **활동 그래프 (Snake Animation)**: 커밋 활동을 시각적으로 표현합니다.

---

## 1단계: 프로필 전용 저장소 생성하기

가장 먼저 GitHub 프로필을 활성화하기 위한 특수 저장소를 생성해야 합니다.

1. GitHub에 로그인한 후, 우측 상단의 **+** 버튼을 눌러 **New repository**를 선택해주세요.
2. 저장소 이름(Repository name)에 본인의 **GitHub 사용자 이름**을 정확하게 입력합니다.
    * 예: 사용자 이름이 `hong-gildong`이라면 저장소 이름도 `hong-gildong`이어야 합니다.
3. **Public**으로 공개 설정을 선택해주세요.
4. **Add a README file** 옵션을 체크하여 초기화합니다.
5. **Create repository** 버튼을 눌러 생성을 완료합니다.

## 2단계: 기본 레이아웃 적용하기

제공된 템플릿을 사용하여 프로필의 뼈대를 잡는 과정입니다.

1. `docs/profile-kit/templates/README_TEMPLATE.md` 파일을 열어 전체 내용을 복사합니다.
2. 방금 생성한 본인의 프로필 저장소에서 `README.md` 파일을 편집 모드로 엽니다.
3. 기존 내용을 모두 지우고 복사한 템플릿 내용을 붙여넣어 주세요.
4. 다음 항목들을 본인에게 맞게 수정합니다.
    * **Typing SVG 설정**: 상단 이미지 태그 내의 `lines=` 부분을 찾아 본인의 직무나 키워드로 변경합니다 (예: Backend Developer, Java Expert 등).
    * **기술 아이콘**: `Bento Grid` 섹션의 `skillicons.dev` 주소에 포함된 기술명(react, java 등)을 본인이 사용하는 기술로 변경합니다.
    * **사용자 이름**: `YOUR_USERNAME`이라고 적힌 부분을 모두 찾아 본인의 실제 GitHub ID로 교체해주세요. (GitHub Stats 카드 및 Snake 애니메이션 이미지 경로)

## 3단계: 기술 스택 및 기여 내역 작성하기

AI를 활용하여 본인의 경험을 전문적인 용어로 정리하는 단계입니다.

1. `docs/profile-kit/AI_PROMPT.md` 파일을 열어 프롬프트 전문을 복사합니다.
2. ChatGPT나 Claude와 같은 AI 챗봇에 복사한 프롬프트를 붙여넣습니다.
3. 프롬프트 하단의 **[PASTE YOUR RAW DATA HERE]** 부분에 본인의 이력서 요약이나 프로젝트 경험, 혹은 작업했던 git 로그 등을 입력하고 전송합니다.
4. AI가 생성해 준 마크다운(Markdown) 형식을 복사합니다.
5. 프로필 `README.md`의 **Engineering Tech Stack & Contributions** 섹션 아래에 붙여넣습니다.

## 4단계: 활동 그래프 애니메이션 설정하기

잔디 심기 활동을 뱀 게임 형태로 보여주는 애니메이션을 추가해 보겠습니다.

1. 본인의 프로필 저장소에서 `.github/workflows` 경로로 폴더를 생성합니다. (이미 있다면 건너뜁니다.)
2. 해당 폴더 안에 `snake.yml`이라는 새 파일을 생성합니다.
3. `docs/profile-kit/templates/snake.yml` 파일의 내용을 그대로 복사하여 붙여넣습니다.
4. 파일을 커밋(Commit)하여 저장합니다.
5. **Actions** 탭으로 이동하여 `Generate Snake` 워크플로우가 보이는지 확인합니다.
    * 이 작업은 매일 자정에 자동으로 실행되지만, 처음에는 수동으로 실행해 보는 것이 좋습니다. `Run workflow` 버튼을 눌러 실행해주세요.
6. 작업이 성공적으로 끝나면, `README.md` 하단에 뱀 게임 이미지가 정상적으로 나타나는지 확인합니다.

## 마무리 및 확인 사항

모든 설정이 완료되었습니다. 프로필 페이지로 이동하여 다음 사항들을 점검해 보시기 바랍니다.

* 상단 배너의 글자가 정상적으로 타이핑되는지 확인합니다.
* 기술 스택 아이콘이 본인의 기술과 일치하는지 확인합니다.
* 기여 내역의 링크들이 올바른 저장소로 연결되는지 확인합니다.
* GitHub Stats와 Snake 애니메이션 이미지가 깨지지 않고 잘 나오는지 확인합니다.

작성하시느라 고생 많으셨습니다. 멋진 프로필이 완성되기를 응원합니다.
